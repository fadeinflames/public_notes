# SRE

This section is the main map for SRE material: incident response, observability, reliability practices, platforms, and operational references.

## Suggested structure

- Incident response: triage, severity, communication, postmortems.
- Observability: logs, metrics, traces, SLOs, dashboards, alerting.
- Reliability practices: error budgets, capacity, load testing, release safety.
- Platforms: Kubernetes, cloud, CI/CD, infrastructure.
- Fundamentals: Linux, networking, databases, distributed systems.

## Contribution rule

Prefer concrete operational pages over broad essays. A short checklist that helps during an incident is more valuable than a long note nobody can scan under pressure.
